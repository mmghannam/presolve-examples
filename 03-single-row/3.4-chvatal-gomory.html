Presolve Reductions in MIP — Worked Examples

3.4 Chvátal–Gomory strengthening

Paper §3.4

The reduction

Take a ≥-inequality whose support consists entirely of non-negative integer variables: Ai⋅​x≥bi​,xj​∈Z≥0​ for all j∈supp(Ai⋅​). (If some coefficients are negative, complement the corresponding variables. If some lower bounds are non-zero, shift the variables so they are. The "all coefficients non-negative, all lower bounds zero" form is canonical for this reduction.)

For any positive scalar s>0, the Chvátal–Gomory cut is ⌈sAi⋅​⌉x≥⌈sbi​⌉.(CGs​) This is always valid for the integer-feasible region. Sometimes it is strictly tighter than the original constraint in the LP relaxation — in which case we replace the original with the CG cut.

Why the CG cut is valid

Two inequalities chained together:

  1. Multiply both sides of the original by s>0 (preserves the inequality direction): sAi⋅​x≥sbi​.
  2. Each coefficient satisfies saij​≤⌈saij​⌉, and multiplying by xj​≥0 preserves the inequality direction. Summing over j, sAi⋅​x≤⌈sAi⋅​⌉x. Combining: ⌈sAi⋅​⌉x≥sbi​.

Finally, the left-hand side is a sum of products of integer coefficients and integer variables, so it is itself integer. Any integer at least sbi​ is at least ⌈sbi​⌉. So ⌈sAi⋅​⌉x≥⌈sbi​⌉.

The two integrality assumptions — integer variables and non-negative bounds — are both essential. Drop either one and at least one step of the chain fails.

When the CG cut strictly tightens the LP

The CG cut and the original constraint compare on equal footing once we normalize both to the same RHS. Dividing (CGs​) through by ⌈sbi​⌉/bi​ gives the same RHS as the original; its coefficients become ⌈sbi​⌉⌈saij​⌉⋅bi​​. The CG cut dominates the original in the LP if and only if ⌈sbi​⌉⌈saij​⌉⋅bi​​≤aij​for every j,with strict inequality for some k. (Smaller coefficients on a ≥-inequality with x≥0 mean a tighter constraint.)

Choosing s

Picking s is heuristic: there is no closed-form choice. A typical implementation tries a small fixed set of candidates — for instance s∈{1,t/amax​,t/amin​,(2t−1)/(2amin​):t=1,…,5} where amax​,amin​ are the largest and smallest non-zero coefficients. If none of them produce a strict tightening, the row is left alone.

Instance

\
\ Three binary variables with distinct coefficients 6, 5, 4 on the left and
\ RHS 7 on the right. Each variable alone (at value 1) is too small to
\ satisfy the constraint, so any feasible point must have at least two of
\ the three variables set to 1.
\
\ The Chvátal–Gomory cut with scalar s = 1/6 captures this directly:
\   x + y + z >= 2

Minimize
 obj: x + y + z
Subject To
 c1: 6 x + 5 y + 4 z >= 7
Bounds
 0 <= x <= 1
 0 <= y <= 1
 0 <= z <= 1
General
 x y z
End

The combinatorial content: each variable alone, at value 1, contributes at most 6 to the row — strictly less than bi​=7. So at least two of x,y,z must be 1 in any feasible point. This "covering" structure is what the CG cut is going to capture.

What fires

Try s=1/6 (this corresponds to t/amax​ with t=1 in the heuristic set above).

  • s⋅aij​ for j=x,y,z is 1, 5/6, 4/6, all of which round up to 1. So ⌈sAi⋅​⌉=(1,1,1).
  • s⋅bi​=7/6, which rounds up to 2. So ⌈sbi​⌉=2.

The candidate CG cut is x+y+z≥2.

Dominance check — normalize the cut to RHS 7 by multiplying through by 7/2: 27​x+27​y+27​z≥7. Compare coefficient-by-coefficient against the original (6,5,4):

All three are strictly less. The dominance condition holds, so the original row is replaced by the CG cut.

The LP relaxation is genuinely tighter. For example, (x,y,z)=(1,0,0.5) was LP-feasible originally (6+0+2=8≥7) and is now cut off (1.5<2). And (1,0,0), (0,1,0), (0,0,1) — the single-variable activations that satisfied the original constraint's LP relaxation but never any integer feasible solution — are all cut off as well.

Notes

  • Integrality is essential. Both pieces — integer variables and non-negative lower bounds — are needed to round both the LHS up to its ceiling and the RHS up to its ceiling. For a continuous variable, saij​xj​≤⌈saij​⌉xj​ still holds, but the LHS of the resulting inequality is no longer integer, so the RHS cannot be ceilinged.
  • Special case: s integer. If s is itself an integer, then ⌈saij​⌉=saij​ and ⌈sbi​⌉=sbi​, so the CG cut is just the original multiplied by s — no improvement. Useful values of s are fractional.
  • Relation to general Chvátal–Gomory cuts. The full CG procedure combines a non-negative linear combination of multiple constraints before rounding, which is how CG cuts are generated in branch-and-cut. The presolve version is the simplest case: one row, one multiplier. The combinatorial reach of the cut is correspondingly more modest, but it is also far cheaper to find.
