Presolve Reductions in MIP — Worked Examples

3.1 Model cleanup and removal of redundant constraints

Paper §3.1

The reduction

Recall (see notation) the row activity bounds inf{Ai⋅​x}=j:aij​>0∑​aij​ℓj​+j:aij​<0∑​aij​uj​, sup{Ai⋅​x}=j:aij​>0∑​aij​uj​+j:aij​<0∑​aij​ℓj​. Every feasible point assigns row i an activity somewhere in [inf,sup].

For a ≤-inequality Ai⋅​x≤bi​, two outcomes are immediate from these bounds:

  • if sup{Ai⋅​x}≤bi​+ε, the row is redundant — every feasible point already satisfies it, so the row can be dropped without changing the feasible region;
  • if inf{Ai⋅​x}>bi​+ε, the row is infeasible — no feasible point exists, so the whole model is infeasible from this single row.

For an equation Ai⋅​x=bi​ both checks apply: redundant if inf≥bi​−ε and sup≤bi​+ε; infeasible if either strict inequality fails.

The check looks only at the row's own coefficients and the global variable bounds — no interaction with other constraints. Multi-row redundancy (a row implied by several other rows working together) is a separate, more expensive reduction; see §5.1.

Despite its simplicity, this reduction fires very often: the paper reports that on its test set, 2525 out of 3182 models contain at least one row that can be classified as redundant using only the original variable bounds.

Instance

\ §3.1 Model cleanup and removal of redundant constraints
\
\ Constraint c2 is redundant: sup{x + y} over the bounds is 10 + 10 = 20,
\ well below the right-hand side of 100. The constraint can be dropped
\ without changing the feasible region.

Minimize
 obj: x + 2 y
Subject To
 c1: x + y >= 3
 c2: x + y <= 100
Bounds
 0 <= x <= 10
 0 <= y <= 10
End

What fires

Compute the activity bounds of c2. The support of c2 is {x,y} with both coefficients positive (ax​=ay​=1), so by the formula above sup{c2​}=1⋅ux​+1⋅uy​=10+10=20. The right-hand side is 100, and 20≤100+ε. So c2 is redundant — every feasible point already satisfies it — and the row is dropped.

For comparison, c1 has inf{c1​}=0 and sup{c1​}=20 against right-hand side 3. Neither 0>3 (infeasibility) nor 20<3 (redundancy for ≥) holds, so the row stays.

Post-presolve model

\ After dropping the redundant constraint c2.
\ The remaining model is the same LP without that row.

Minimize
 obj: x + 2 y
Subject To
 c1: x + y >= 3
Bounds
 0 <= x <= 10
 0 <= y <= 10
End

The model shrinks from two constraints to one. The optimal solution is unchanged: x⋆=3, y⋆=0 with objective value 3.

Notes

  • Infeasibility detection is the same reduction, run the other way. If the right-hand side of c2 were −1 instead of 100, then inf{c2​}=0>−1+ε would prove the model infeasible from this single row, with no other reasoning required.
  • Coefficient cleanup. A secondary flavor of "cleanup" zeros coefficients whose magnitude is so small that their largest possible contribution to row activity is bounded by a fraction of the feasibility tolerance. The model is mathematically unchanged within tolerance — the row's activity bounds shift by at most ε — but the constraint matrix gets sparser and better scaled, which speeds up downstream LP solves and reduces round-off in the simplex factorization. The thresholds are numerical and depend on the implementation; the idea is just "if a coefficient cannot move the row activity by more than a noise-level amount, treat it as zero."
  • Why it matters. Redundant rows do nothing for the LP relaxation but they still cost time in every iteration of the simplex method, every cut-separation pass, and every node propagation step. Each individual row removal is a small constant-factor speed-up; the cumulative effect on real models is large.
