Presolve Reductions in MIP — Worked Examples

3.2 Bound strengthening

Paper §3.2

The reduction

Take a single inequality Ai⋅​x≤bi​ and pick one variable xk​ in its support. Split the row into "xk​ alone" and "everything else": AiS​xS​+aik​xk​≤bi​,S=supp(Ai⋅​)∖{k}. The activity of the "everything else" block over its bounds is, at minimum, ℓiS​:=inf{AiS​xS​}=j∈S, aij​>0∑​aij​ℓj​+j∈S, aij​<0∑​aij​uj​. Whatever happens elsewhere in the row, AiS​xS​ is at least ℓiS​, so xk​ can contribute at most bi​−ℓiS​ to the row's activity. Dividing by aik​ gives a bound on xk​:

  • if aik​>0, then xk​≤(bi​−ℓiS​)/aik​, tightening the upper bound: uk​:=min{uk​, aik​bi​−ℓiS​​};
  • if aik​<0, then xk​≥(bi​−ℓiS​)/aik​ (the inequality flips because we divided by a negative), tightening the lower bound: ℓk​:=max{ℓk​, aik​bi​−ℓiS​​}.

For integer xk​ the new bound is rounded inward (⌊⋅⌋ for an upper bound, ⌈⋅⌉ for a lower bound), since fractional values aren't achievable anyway.

The check requires ℓiS​ to be finite; if any j∈S has ℓj​=−∞ on a positive coefficient (or uj​=+∞ on a negative one), then ℓiS​=−∞ and the formula gives no information.

This is the simplest form of domain propagation — the workhorse of every constraint propagation engine.

Instance

\ §3.2 Bound strengthening
\
\ The constraint 4x + 3y <= 15 implies an upper bound on x:
\   4x <= 15 - 3 * inf{y} = 15 - 0 = 15
\   x <= 15 / 4 = 3.75
\ Since x is integer, the bound rounds inward to u_x = 3.

Minimize
 obj: x + y
Subject To
 c1: 4 x + 3 y <= 15
Bounds
 0 <= x <= 10
 0 <= y <= 5
General
 x
End

What fires

Apply the formula to constraint c1 with k=x.

  • Set S={y}, so AiS​=(3).
  • Coefficient sign: ay​=3>0, contribution to the lower activity is 3⋅ℓy​=3⋅0=0. Therefore ℓiS​=0.
  • Coefficient on the target variable: ax​=4>0, so we tighten an upper bound: ux​≤415−0​=3.75.
  • x is integer, so round inward: ux​:=⌊3.75⌋=3.

The variable's upper bound moves from 10 to 3.

Repeating the formula for k=y (just to confirm there is nothing more to do): uy​≤315−0​=5, which matches the existing upper bound uy​=5. No change.

The constraint itself is unchanged; only x's upper bound has been tightened.

Notes

  • Cascading. A single round of bound strengthening can trigger more in the next round: tightening ux​ makes some other row's ℓiS​ tighter (because ux​ entered with a negative coefficient, say), which then tightens some third variable's bound, and so on. Practical implementations iterate the procedure across all rows until no further changes are observed.
  • Infinite chains are possible on continuous models. The paper gives the example x1​−ax2​=0,ax1​−x2​=0,0≤x1​,x2​≤1,0<a<1. Starting from ux1​​=1, the first equation tightens ux1​​≤a, the second then gives ux2​​≤a2, the first then gives ux1​​≤a3, and so on — an infinite sequence of strictly smaller bounds converging to 0, but never finishing. Practical implementations guard against this by capping the number of passes, requiring a minimum step size on each update, and bounding the absolute magnitude of any new bound. Integer models do not have this issue because each tightening of an integer bound moves it by at least one.
  • Infeasibility as a side effect. If a derived upper bound is less than the current lower bound (or vice versa), the model is infeasible at this constraint.
