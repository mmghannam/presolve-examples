Presolve Reductions in MIP — Worked Examples

3.5 Euclidean and modular inverse reduction

Paper §3.5

This section is really two reductions stacked on top of each other. The first — the Euclidean reduction — is simple. The second — the modular inverse substitution — needs a short detour through modular arithmetic before it lands cleanly.

The Euclidean reduction

Given a constraint with all-integer variables, j=1∑n​aij​xj​∘i​bi​,xj​∈Z, let d=gcd(ai1​,…,ain​) in the extended sense (the largest rational such that every aij​/d is an integer). Both sides of the constraint are multiples of d, so we can divide through: j=1∑n​daij​​xj​∘i​⌊dbi​​⌋. For an equation, infeasibility is detected immediately if bi​/d∈/Z: the left-hand side is an integer multiple of d but the right-hand side is not.

Example: 4x1​+6x2​=10 has d=gcd(4,6)=2, and we divide through to 2x1​+3x2​=5.

This is the Euclidean reduction — straightforward, fires whenever the row's coefficients share a common factor.

When the Euclidean reduction does nothing

What if the coefficients are already coprime — gcd=1? Then we cannot divide through and the Euclidean reduction is a no-op. But integer structure can still be hidden in the equation. Consider 1867x1​+1913x2​=3,618,894,x1​,x2​∈Z≥0​. The coefficients 1867 and 1913 are coprime (both are prime), so the Euclidean reduction has nothing to say. And yet we are about to discover that x1​ is forced to be of a very specific form.

The trick: reduce both sides of the equation modulo 1913, the coefficient of x2​. The 1913x2​ term vanishes (it's a multiple of 1913), leaving 1867x1​≡3,618,894(mod1913). Since 3,618,894=1891⋅1913+1411, the right-hand side reduces to 1411(mod1913), giving 1867x1​≡1411(mod1913). This is a one-variable congruence. If we could "divide both sides by 1867" in modular arithmetic, we would learn exactly what residue x1​ must take modulo 1913.

That "division" is the modular inverse.

Modular inverse, defined

In ordinary arithmetic, the inverse of a number a is the number a−1 such that a⋅a−1=1. In modular arithmetic the analogous definition is:

The modular multiplicative inverse of a modulo m is the integer a−1∣m∈{1,2,…,m−1} satisfying a⋅a−1∣m≡1(modm).

Read "a−1∣m" as "the inverse of a, mod m." It exists if and only if gcd(a,m)=1.

To check what the inverse means: in arithmetic mod 7, 3⋅5=15=2⋅7+1≡1(mod7), so 3−1∣7=5. Then to solve 3x≡4(mod7) we just multiply both sides by 5: x≡20≡6(mod7). The inverse "undoes" multiplication by a, exactly like ordinary division.

For our instance, 1867−1∣1913=1206 (computed once and for all using the extended Euclidean algorithm). Verifying: 1867⋅1206=2,251,602=1177⋅1913+1≡1(mod1913).✓

The modular inverse reduction

Equipped with the modular inverse, return to the congruence 1867x1​≡1411(mod1913) and multiply both sides by 1867−1∣1913=1206: x1​≡1206⋅1411(mod1913). Computing: 1206⋅1411=1,701,666=889⋅1913+1009, so x1​≡1009(mod1913).

So every feasible x1​ has the form x1​=1913z+1009for some z∈Z≥0​. This is an affine substitution. We replace x1​ throughout the model by 1913z+1009, where z is a fresh integer variable.

The same trick is applicable whenever an equation contains an integer variable x1​ whose coefficient ai1​ is coprime to dˉ:=gcd(ai2​,…,ain​), with dˉ≥2. The substitution is x1​:=dˉ⋅z+bˉ,z∈Z,bˉ≡ai1−1∣dˉ​⋅bi​(moddˉ). After substituting back, the equation's new coefficients all share the factor dˉ, so the (ordinary, non-modular) Euclidean reduction fires on the next pass.

What fires

We just did the math. Substitute x1​:=1913z+1009 back into the original equation: 1867⋅(1913z+1009)+1913x2​=3,618,894⟹3,571,571z+1913x2​=1,735,091. Now the coefficients 3,571,571=1867⋅1913 and 1913 share gcd=1913 — the Euclidean reduction now fires on the transformed row, dividing through to 1867z+x2​=907.

Post-presolve model

\ After the modular-inverse substitution x1 := 1913 z + 1009.
\
\ Substituting into the original equation gives 3571571 z + 1913 x2 = 1735091,
\ which divides through by gcd 1913 to yield 1867 z + x2 = 907.
\
\ Objective: x1 + x2 = (1913 z + 1009) + x2 = 1913 z + x2 + 1009.
\ The trailing +1009 is the objective constant (uncrush offset for x1).

Minimize
 obj: 1913 z + x2 + 1009
Subject To
 c1: 1867 z + x2 = 907
Bounds
 z >= 0
 x2 >= 0
General
 z x2
End

The reduced model has the unique optimal solution z=0, x2​=907 — since 1867z≤907 forces z=0. Uncrushing: x1​=1913⋅0+1009=1009, with optimal objective value 1009+907=1916.

Without this reduction, branch-and-bound on the original equation has to enumerate many candidate values of x1​ before stumbling on 1009 — the original instance has no useful LP structure to guide the search. The modular inverse compresses that combinatorial work into a single substitution at the root.

Notes

  • Where modular inverses come from. The extended Euclidean algorithm computes gcd(a,m) and, as a by-product, two integers s,t with sa+tm=gcd(a,m). When gcd=1, that says sa≡1(modm), so a−1∣m=smodm. This is exactly the same algorithm the Euclidean reduction itself uses, just instrumented to track the coefficients.
  • When the modular trick is even possible. The substitution requires that ai1​ have an absolute value distinct from every other coefficient in the row. If two coefficients share an absolute value, then dˉ=gcd(ai2​,…,ain​)=1 trivially, and no reduction follows.
  • One continuous variable is allowed. The reduction works even if exactly one variable in the equation is continuous, provided that variable plays the role of x1​ — its value is then locked to a deterministic affine function of an integer auxiliary, which downstream presolves can propagate.
  • Fractional coefficients. When the row's coefficients are rational rather than integral, the gcd has to be defined in the extended rational sense above. In practice this is approximated heuristically, since computing exact rational gcds in finite-precision arithmetic is brittle.
