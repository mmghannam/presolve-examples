# 5.1 Redundancy detection

> Paper §5.1

A constraint $r$ is **redundant** if removing it does not change the feasible set of the model. Deciding redundancy in full generality is as hard as the model itself (NP-complete for MIP), but several restricted variants are polynomial:

- **Single-row redundancy** — a row whose activity bounds already imply it. Already covered in §3.1.
- **LP-redundancy from one auxiliary row** — for a candidate row $r$, checking if $\sup\{A_{r\cdot}x : A_{S \cdot}x \circ_S b_S\} \leq b_r$ on a subset $S$ that excludes $r$. This is an LP, polynomial in size, and a useful sweet spot.
- **Dependent equations** — a set of equations $\{A_{i\cdot}x = b_i\}$ may contain a row that is a linear combination of the others. A rank-revealing LU factorization detects such rows; a redundant equation $0 = 0$ is removed, and an inconsistent reduction $0 = b$ with $b \neq 0$ proves infeasibility.
- **Inequality subsumption** — an inequality $A_{S}x \leq b'$ is subsumed by $A_{S}x + d y \leq b''$ if the activity of $dy$ is provably $\geq b'' - b'$. The subsumed row can be discarded as it never cuts more than the subsumer.
- **Set-packing / clique subsumption** — a set-packing constraint is dominated if every variable in it appears in a larger clique inequality elsewhere in the model.
