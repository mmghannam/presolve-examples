# 7.1 Aggregate pairs of symmetric variables

> Paper §7.1

A permutation $\pi: N \to N$ of the variable indices is a **symmetry generator** of the model if relabeling the columns by $\pi$ (and possibly the rows by some companion row permutation $\sigma$) leaves the model unchanged: same objective coefficients on permuted indices, same bounds, same integrality, same $A$ entries up to $(\sigma, \pi)$.

If $\pi$ has the additional property that no two distinct variables in the same orbit appear in any row together — formally, $\operatorname{supp}(A_{\cdot j}) \cap \operatorname{supp}(A_{\cdot \pi(j)}) = \emptyset$ for every $j$ moved by $\pi$ — then $\pi$ is **non-overlapping**. For non-overlapping symmetry generators the orbit's variables are interchangeable in every constraint and can be **aggregated**: replace every variable in an orbit with a single representative $x_j$.

For *continuous-only* symmetry generators (those that move only continuous variables), aggregation is also valid; the aggregated variable represents the arithmetic mean of the orbit. This averaging step does not preserve integrality, which is why integer orbits require the non-overlap property.

Paper Example 9 — a small binary IP with two pairs of symmetric variables — illustrates the aggregation cleanly.

Symmetry detection itself is a graph-isomorphism problem on the row/column incidence graph and is solved using off-the-shelf tools.
