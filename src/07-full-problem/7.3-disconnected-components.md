# 7.3 Disconnected components

> Paper §7.3

Build a graph on the variables where two variables are connected if they appear together in some constraint. The connected components of this graph correspond to **independent sub-MIPs**: if $N$ partitions into $U \cup V$ with no constraint touching variables from both, then
$$\min\{c_U^T x_U + c_V^T x_V : A_{RU} x_U \leq b_R,\ A_{KV} x_V \leq b_K\}$$
decomposes into
$$\min\{c_U^T x_U : A_{RU} x_U \leq b_R\} + \min\{c_V^T x_V : A_{KV} x_V \leq b_K\}.$$

Two practical wrinkles complicate the simple "solve each separately" approach:

1. **No incumbent until the last component finishes.** Solving components serially means the first complete feasible solution to the full model is only available after every component but the last has been solved. For long-running components this is undesirable.
2. **Gap limits don't decompose.** A 0.01% optimality-gap target on the *full* objective doesn't translate cleanly to per-component gap targets, so any nonzero gap setting requires careful book-keeping.

A common compromise is to solve all but the largest component to true optimality during presolve (with strict work limits as protection against pathologically hard small components), fix their variables in the master, and then solve the residual in the normal MIP loop. If a component is too hard, the procedure aborts and falls back to an interleaved per-component search.

Disconnected components are sometimes intrinsic to the original formulation but more often arise *after* presolve has aggregated, fixed, or eliminated variables in the larger model.
