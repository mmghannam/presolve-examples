# 8.1 Bound strengthening at nodes

> **DRAFT — unreviewed.** This chapter is a scaffolded starting draft. The math, instance, and prose have not been reviewed. See the §3.5 and §4.5 chapters elsewhere in the book for the target format.

> Paper §8.1

Bound strengthening at the root (§3.2 / §5.4) tightens variable bounds globally; **node bound strengthening** does the same locally, using only the *current* node's branching restrictions.

The trigger is a branching decision: when the search creates a child node by adding $x_b \leq \bar u_b$ or $x_b \geq \bar\ell_b$, every constraint that contains $x_b$ has its activity bounds altered. The §3.2 propagation rules then derive new local bounds on the variables sharing those constraints, which in turn alter activities on *other* constraints, and the process iterates.

The reductions cascade: a single branching decision can trigger many local fixings and substitutions in the subtree. The effect is cumulative across the search — every node refines the bounds of its parent — and the cost is amortized by only revisiting constraints whose activity bounds actually changed since the parent.

Because node presolve runs at every search node, its per-call cost dominates correctness: the propagation passes are capped (typically a small fixed number of rounds), and only constraints that contain a recently-changed variable are revisited.
