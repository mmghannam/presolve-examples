# 3.4 Chvátal–Gomory strengthening

> Paper §3.4

## The reduction

Take a $\geq$-inequality whose support consists entirely of **non-negative integer** variables:
$$A_{i\cdot}\,x \geq b_i, \qquad x_j \in \mathbb{Z}_{\geq 0} \text{ for all } j \in \operatorname{supp}(A_{i\cdot}).$$
(If some coefficients are negative, complement the corresponding variables. If some lower bounds are non-zero, shift the variables so they are. The "all coefficients non-negative, all lower bounds zero" form is canonical for this reduction.)

For any positive scalar $s > 0$, the **Chvátal–Gomory cut** is
$$\lceil s\,A_{i\cdot} \rceil\,x \geq \lceil s\,b_i \rceil. \tag{CG$_s$}$$
This is always valid for the integer-feasible region. Sometimes it is *strictly tighter* than the original constraint in the LP relaxation — in which case we replace the original with the CG cut.

### Why the CG cut is valid

Two inequalities chained together:

1. Multiply both sides of the original by $s > 0$ (preserves the inequality direction): $s\,A_{i\cdot}\,x \geq s\,b_i$.
2. Each coefficient satisfies $s\,a_{ij} \leq \lceil s\,a_{ij} \rceil$, and multiplying by $x_j \geq 0$ preserves the inequality direction. Summing over $j$,
$$s\,A_{i\cdot}\,x \leq \lceil s\,A_{i\cdot} \rceil\,x.$$
Combining: $\lceil s\,A_{i\cdot} \rceil\,x \geq s\,b_i$.

Finally, the left-hand side is a sum of products of *integer* coefficients and *integer* variables, so it is itself integer. Any integer at least $s\,b_i$ is at least $\lceil s\,b_i \rceil$. So
$$\lceil s\,A_{i\cdot} \rceil\,x \geq \lceil s\,b_i \rceil.$$

The two integrality assumptions — integer variables and non-negative bounds — are both essential. Drop either one and at least one step of the chain fails.

### When the CG cut strictly tightens the LP

The CG cut and the original constraint compare on equal footing once we normalize both to the same RHS. Dividing $(\text{CG}_s)$ through by $\lceil s\,b_i \rceil / b_i$ gives the same RHS as the original; its coefficients become
$$\frac{\lceil s\,a_{ij} \rceil \cdot b_i}{\lceil s\,b_i \rceil}.$$
The CG cut dominates the original in the LP if and only if
$$\frac{\lceil s\,a_{ij} \rceil \cdot b_i}{\lceil s\,b_i \rceil} \leq a_{ij} \quad \text{for every } j, \qquad \text{with strict inequality for some } k.$$
(Smaller coefficients on a $\geq$-inequality with $x \geq 0$ mean a tighter constraint.)

### Choosing $s$

Picking $s$ is heuristic: there is no closed-form choice. A typical implementation tries a small fixed set of candidates — for instance $s \in \{1, t/a_{\max}, t/a_{\min}, (2t-1)/(2 a_{\min}) : t = 1, \ldots, 5\}$ where $a_{\max}, a_{\min}$ are the largest and smallest non-zero coefficients. If none of them produce a strict tightening, the row is left alone.

The combinatorial content: each variable alone, at value 1, contributes at most $6$ to the row — strictly less than $b_i = 7$. So at least *two* of $x, y, z$ must be 1 in any feasible point. This "covering" structure is what the CG cut is going to capture.

## What fires

Try $s = 1/6$ (this corresponds to $t/a_{\max}$ with $t = 1$ in the heuristic set above).

- $s \cdot a_{ij}$ for $j = x, y, z$ is $1,\ 5/6,\ 4/6$, all of which round up to $1$. So $\lceil s\,A_{i\cdot} \rceil = (1, 1, 1)$.
- $s \cdot b_i = 7/6$, which rounds up to $2$. So $\lceil s\,b_i \rceil = 2$.

The candidate CG cut is
$$x + y + z \geq 2.$$

Dominance check — normalize the cut to RHS $7$ by multiplying through by $7/2$:
$$\tfrac{7}{2}\,x + \tfrac{7}{2}\,y + \tfrac{7}{2}\,z \geq 7.$$
Compare coefficient-by-coefficient against the original $(6, 5, 4)$:

| $j$ | Original $a_{ij}$ | Normalized CG coefficient | Strict? |
|---|---|---|---|
| $x$ | $6$ | $3.5$ | ✓ |
| $y$ | $5$ | $3.5$ | ✓ |
| $z$ | $4$ | $3.5$ | ✓ |

All three are strictly less. The dominance condition holds, so the original row is replaced by the CG cut.

## Notes

- **Integrality is essential.** Both pieces — integer variables and non-negative lower bounds — are needed to round both the LHS up to its ceiling and the RHS up to its ceiling. For a continuous variable, $s a_{ij} x_j \leq \lceil s a_{ij} \rceil x_j$ still holds, but the LHS of the resulting inequality is no longer integer, so the RHS cannot be ceilinged.
- **Special case: $s$ integer.** If $s$ is itself an integer, then $\lceil s\,a_{ij} \rceil = s\,a_{ij}$ and $\lceil s\,b_i \rceil = s\,b_i$, so the CG cut is just the original multiplied by $s$ — no improvement. Useful values of $s$ are fractional.
- **Relation to general Chvátal–Gomory cuts.** The full CG procedure combines a non-negative linear combination of *multiple* constraints before rounding, which is how CG cuts are generated in branch-and-cut. The presolve version is the simplest case: one row, one multiplier. The combinatorial reach of the cut is correspondingly more modest, but it is also far cheaper to find.
