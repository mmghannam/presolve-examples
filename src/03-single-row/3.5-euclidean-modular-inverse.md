# 3.5 Euclidean and modular inverse reduction

> Paper §3.5

This section is really two reductions stacked on top of each other. The first — the Euclidean reduction — is simple. The second — the modular inverse substitution — needs a short detour through modular arithmetic before it lands cleanly.

## The Euclidean reduction

Given a constraint with all-integer variables,
$$\sum_{j=1}^{n} a_{ij} x_j \circ_i b_i, \qquad x_j \in \mathbb{Z},$$
let $d = \gcd(a_{i1}, \ldots, a_{in})$ in the extended sense (the largest rational such that every $a_{ij}/d$ is an integer). Both sides of the constraint are multiples of $d$, so we can divide through:
$$\sum_{j=1}^{n} \frac{a_{ij}}{d}\, x_j \circ_i \left\lfloor \frac{b_i}{d} \right\rfloor.$$
For an equation, infeasibility is detected immediately if $b_i/d \notin \mathbb{Z}$: the left-hand side is an integer multiple of $d$ but the right-hand side is not.

Example: $4x_1 + 6x_2 = 10$ has $d = \gcd(4, 6) = 2$, and we divide through to $2x_1 + 3x_2 = 5$.

This is the *Euclidean* reduction — straightforward, fires whenever the row's coefficients share a common factor.

## When the Euclidean reduction does nothing

What if the coefficients are already coprime — $\gcd = 1$? Then we cannot divide through and the Euclidean reduction is a no-op. But integer structure can still be hidden in the equation. Consider
$$1867\,x_1 + 1913\,x_2 = 3{,}618{,}894, \qquad x_1, x_2 \in \mathbb{Z}_{\geq 0}.$$
The coefficients $1867$ and $1913$ are coprime (both are prime), so the Euclidean reduction has nothing to say. And yet we are about to discover that $x_1$ is forced to be of a very specific form.

The trick: reduce both sides of the equation **modulo $1913$**, the coefficient of $x_2$. The $1913\,x_2$ term vanishes (it's a multiple of $1913$), leaving
$$1867\,x_1 \equiv 3{,}618{,}894 \pmod{1913}.$$
Since $3{,}618{,}894 = 1891 \cdot 1913 + 1411$, the right-hand side reduces to $1411 \pmod{1913}$, giving
$$1867\,x_1 \equiv 1411 \pmod{1913}.$$
This is a one-variable congruence. If we could "divide both sides by $1867$" in modular arithmetic, we would learn exactly what residue $x_1$ must take modulo $1913$.

That "division" is the **modular inverse**.

## Modular inverse, defined

In ordinary arithmetic, the inverse of a number $a$ is the number $a^{-1}$ such that $a \cdot a^{-1} = 1$. In **modular arithmetic** the analogous definition is:

> The **modular multiplicative inverse** of $a$ modulo $m$ is the integer $a^{-1|m} \in \{1, 2, \ldots, m-1\}$ satisfying
> $$a \cdot a^{-1|m} \equiv 1 \pmod{m}.$$

Read "$a^{-1|m}$" as "the inverse of $a$, mod $m$." It exists if and only if $\gcd(a, m) = 1$.

To check what the inverse *means*: in arithmetic mod $7$, $3 \cdot 5 = 15 = 2 \cdot 7 + 1 \equiv 1 \pmod 7$, so $3^{-1|7} = 5$. Then to solve $3 x \equiv 4 \pmod 7$ we just multiply both sides by $5$: $x \equiv 20 \equiv 6 \pmod 7$. The inverse "undoes" multiplication by $a$, exactly like ordinary division.

For our instance, $1867^{-1|1913} = 1206$ (computed once and for all using the extended Euclidean algorithm). Verifying:
$$1867 \cdot 1206 = 2{,}251{,}602 = 1177 \cdot 1913 + 1 \equiv 1 \pmod{1913}. \checkmark$$

## The modular inverse reduction

Equipped with the modular inverse, return to the congruence $1867\,x_1 \equiv 1411 \pmod{1913}$ and multiply both sides by $1867^{-1|1913} = 1206$:
$$x_1 \equiv 1206 \cdot 1411 \pmod{1913}.$$
Computing: $1206 \cdot 1411 = 1{,}701{,}666 = 889 \cdot 1913 + 1009$, so
$$x_1 \equiv 1009 \pmod{1913}.$$

So every feasible $x_1$ has the form
$$x_1 = 1913\,z + 1009 \qquad \text{for some } z \in \mathbb{Z}_{\geq 0}.$$
This is an **affine substitution**. We replace $x_1$ throughout the model by $1913 z + 1009$, where $z$ is a fresh integer variable.

The same trick is applicable whenever an equation contains an integer variable $x_1$ whose coefficient $a_{i1}$ is coprime to $\bar d := \gcd(a_{i2}, \ldots, a_{in})$, with $\bar d \geq 2$. The substitution is
$$x_1 := \bar d \cdot z + \bar b, \qquad z \in \mathbb{Z}, \qquad \bar b \equiv a_{i1}^{-1|\bar d} \cdot b_i \pmod{\bar d}.$$
After substituting back, the equation's new coefficients all share the factor $\bar d$, so the (ordinary, non-modular) Euclidean reduction fires on the next pass.

## What fires

We just did the math. Substitute $x_1 := 1913\,z + 1009$ back into the original equation:
$$1867 \cdot (1913\,z + 1009) + 1913\,x_2 = 3{,}618{,}894 \;\Longrightarrow\; 3{,}571{,}571\,z + 1913\,x_2 = 1{,}735{,}091.$$
Now the coefficients $3{,}571{,}571 = 1867 \cdot 1913$ and $1913$ share $\gcd = 1913$ — the *Euclidean* reduction now fires on the transformed row, dividing through to
$$1867\,z + x_2 = 907.$$

The reduced model has the unique optimal solution $z = 0,\ x_2 = 907$ — since $1867\,z \leq 907$ forces $z = 0$. Uncrushing: $x_1 = 1913 \cdot 0 + 1009 = 1009$, with optimal objective value $1009 + 907 = 1916$.

Without this reduction, branch-and-bound on the original equation has to enumerate many candidate values of $x_1$ before stumbling on $1009$ — the original instance has no useful LP structure to guide the search. The modular inverse compresses that combinatorial work into a single substitution at the root.

## Notes

- **Where modular inverses come from.** The extended Euclidean algorithm computes $\gcd(a, m)$ and, as a by-product, two integers $s, t$ with $sa + tm = \gcd(a, m)$. When $\gcd = 1$, that says $sa \equiv 1 \pmod m$, so $a^{-1|m} = s \bmod m$. This is exactly the same algorithm the Euclidean reduction itself uses, just instrumented to track the coefficients.
- **When the modular trick is even possible.** The substitution requires that $a_{i1}$ have an absolute value distinct from every other coefficient in the row. If two coefficients share an absolute value, then $\bar d = \gcd(a_{i2}, \ldots, a_{in}) = 1$ trivially, and no reduction follows.
- **One continuous variable is allowed.** The reduction works even if exactly one variable in the equation is continuous, provided that variable plays the role of $x_1$ — its value is then locked to a deterministic affine function of an integer auxiliary, which downstream presolves can propagate.
- **Fractional coefficients.** When the row's coefficients are rational rather than integral, the gcd has to be defined in the extended rational sense above. In practice this is approximated heuristically, since computing exact rational gcds in finite-precision arithmetic is brittle.
