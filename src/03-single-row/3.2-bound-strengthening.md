# 3.2 Bound strengthening

> Paper §3.2

## The reduction

Take a single inequality $A_{i\cdot}x \leq b_i$ and pick one variable $x_k$ in its support. Split the row into "$x_k$ alone" and "everything else":
$$A_{iS}\,x_S + a_{ik}\,x_k \leq b_i, \qquad S = \operatorname{supp}(A_{i\cdot}) \setminus \{k\}.$$
The activity of the "everything else" block over its bounds is, at minimum,
$$\ell_{iS} := \inf\{A_{iS}\,x_S\} = \sum_{j \in S,\ a_{ij} > 0} a_{ij}\, \ell_j + \sum_{j \in S,\ a_{ij} < 0} a_{ij}\, u_j.$$
Whatever happens elsewhere in the row, $A_{iS}\,x_S$ is at least $\ell_{iS}$, so $x_k$ can contribute at most $b_i - \ell_{iS}$ to the row's activity. Dividing by $a_{ik}$ gives a bound on $x_k$:

- if $a_{ik} > 0$, then $x_k \leq (b_i - \ell_{iS}) / a_{ik}$, tightening the **upper** bound:
$$u_k := \min\!\left\{ u_k,\ \frac{b_i - \ell_{iS}}{a_{ik}} \right\};$$
- if $a_{ik} < 0$, then $x_k \geq (b_i - \ell_{iS}) / a_{ik}$ (the inequality flips because we divided by a negative), tightening the **lower** bound:
$$\ell_k := \max\!\left\{ \ell_k,\ \frac{b_i - \ell_{iS}}{a_{ik}} \right\}.$$

For integer $x_k$ the new bound is rounded **inward** ($\lfloor \cdot \rfloor$ for an upper bound, $\lceil \cdot \rceil$ for a lower bound), since fractional values aren't achievable anyway.

The check requires $\ell_{iS}$ to be finite; if any $j \in S$ has $\ell_j = -\infty$ on a positive coefficient (or $u_j = +\infty$ on a negative one), then $\ell_{iS} = -\infty$ and the formula gives no information.

This is the simplest form of **domain propagation** — the workhorse of every constraint propagation engine.

## What fires

Apply the formula to constraint `c1` with $k = x$.

- Set $S = \{y\}$, so $A_{iS} = (3)$.
- Coefficient sign: $a_y = 3 > 0$, contribution to the lower activity is $3 \cdot \ell_y = 3 \cdot 0 = 0$. Therefore $\ell_{iS} = 0$.
- Coefficient on the target variable: $a_x = 4 > 0$, so we tighten an upper bound:
  $$u_x \leq \frac{15 - 0}{4} = 3.75.$$
- $x$ is integer, so round inward: $u_x := \lfloor 3.75 \rfloor = 3$.

The variable's upper bound moves from $10$ to $3$.

Repeating the formula for $k = y$ (just to confirm there is nothing more to do):
$$u_y \leq \frac{15 - 0}{3} = 5,$$
which matches the existing upper bound $u_y = 5$. No change.

## Notes

- **Cascading.** A single round of bound strengthening can trigger more in the next round: tightening $u_x$ makes some other row's $\ell_{iS}$ tighter (because $u_x$ entered with a negative coefficient, say), which then tightens some third variable's bound, and so on. Practical implementations iterate the procedure across all rows until no further changes are observed.
- **Infinite chains are possible on continuous models.** The paper gives the example
$$x_1 - a x_2 = 0, \qquad a x_1 - x_2 = 0, \qquad 0 \leq x_1, x_2 \leq 1,\quad 0 < a < 1.$$
  Starting from $u_{x_1} = 1$, the first equation tightens $u_{x_1} \leq a$, the second then gives $u_{x_2} \leq a^2$, the first then gives $u_{x_1} \leq a^3$, and so on — an infinite sequence of strictly smaller bounds converging to $0$, but never finishing. Practical implementations guard against this by capping the number of passes, requiring a minimum step size on each update, and bounding the absolute magnitude of any new bound. Integer models do not have this issue because each tightening of an integer bound moves it by at least one.
- **Infeasibility as a side effect.** If a derived upper bound is less than the current lower bound (or vice versa), the model is infeasible at this constraint.
