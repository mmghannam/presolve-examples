# 3.1 Model cleanup and removal of redundant constraints

> Paper §3.1

## The reduction

Recall (see [notation](../notation.md)) the row activity bounds
$$\inf\{A_{i\cdot}x\} = \sum_{j : a_{ij} > 0} a_{ij} \ell_j + \sum_{j : a_{ij} < 0} a_{ij} u_j,$$
$$\sup\{A_{i\cdot}x\} = \sum_{j : a_{ij} > 0} a_{ij} u_j + \sum_{j : a_{ij} < 0} a_{ij} \ell_j.$$
Every feasible point assigns row $i$ an activity somewhere in $[\inf,\, \sup]$.

For a $\leq$-inequality $A_{i\cdot}x \leq b_i$, two outcomes are immediate from these bounds:

- if $\sup\{A_{i\cdot}x\} \leq b_i + \varepsilon$, the row is **redundant** — every feasible point already satisfies it, so the row can be dropped without changing the feasible region;
- if $\inf\{A_{i\cdot}x\} > b_i + \varepsilon$, the row is **infeasible** — no feasible point exists, so the whole model is infeasible from this single row.

For an equation $A_{i\cdot}x = b_i$ both checks apply: redundant if $\inf \geq b_i - \varepsilon$ **and** $\sup \leq b_i + \varepsilon$; infeasible if either strict inequality fails.

The check looks only at the row's own coefficients and the global variable bounds — no interaction with other constraints. Multi-row redundancy (a row implied by *several* other rows working together) is a separate, more expensive reduction; see [§5.1](../05-multi-row/5.1-redundancy-detection.md).

Despite its simplicity, this reduction fires very often: the paper reports that on its test set, 2525 out of 3182 models contain at least one row that can be classified as redundant using only the original variable bounds.

## What fires

Compute the activity bounds of `c2`. The support of `c2` is $\{x, y\}$ with both coefficients positive ($a_x = a_y = 1$), so by the formula above
$$\sup\{c_2\} = 1 \cdot u_x + 1 \cdot u_y = 10 + 10 = 20.$$
The right-hand side is $100$, and $20 \leq 100 + \varepsilon$. So `c2` is redundant — every feasible point already satisfies it — and the row is dropped.

For comparison, `c1` has $\inf\{c_1\} = 0$ and $\sup\{c_1\} = 20$ against right-hand side $3$. Neither $0 > 3$ (infeasibility) nor $20 < 3$ (redundancy for $\geq$) holds, so the row stays.

## Notes

- **Infeasibility detection is the same reduction, run the other way.** If the right-hand side of `c2` were $-1$ instead of $100$, then $\inf\{c_2\} = 0 > -1 + \varepsilon$ would prove the model infeasible from this single row, with no other reasoning required.
- **Coefficient cleanup.** A secondary flavor of "cleanup" zeros coefficients whose magnitude is so small that their largest possible contribution to row activity is bounded by a fraction of the feasibility tolerance. The model is mathematically unchanged within tolerance — the row's activity bounds shift by at most $\varepsilon$ — but the constraint matrix gets sparser and better scaled, which speeds up downstream LP solves and reduces round-off in the simplex factorization. The thresholds are numerical and depend on the implementation; the *idea* is just "if a coefficient cannot move the row activity by more than a noise-level amount, treat it as zero."
- **Why it matters.** Redundant rows do nothing for the LP relaxation but they still cost time in every iteration of the simplex method, every cut-separation pass, and every node propagation step. Each individual row removal is a small constant-factor speed-up; the cumulative effect on real models is large.
