Presolve Reductions in MIP — Worked Examples

4.5 Substitute implied free variables

Paper §4.5

The reduction

Let ℓj​,uj​ be the explicit bounds of variable xj​, and let ℓˉj​,uˉj​ be the tightest bounds on xj​ that can be deduced from the other constraints by single-row bound strengthening (§3.2), without using the explicit bounds of xj​ itself. If [ℓˉj​, uˉj​]⊆[ℓj​, uj​], then xj​ is called an implied free variable: its explicit bounds carry no information beyond what the other constraints already enforce. The case ℓj​=−∞, uj​=∞ (a genuinely free variable) is the trivial special case.

If, in addition, there is an equation of the form AiS​xS​+aij​xj​=bi​,S⊆N∖{j}, aij​=0, then xj​ can be substituted out of the entire model: xj​:=aij​bi​−AiS​xS​​. The equation is consumed; xj​ is eliminated from every other row and from the objective. For integer xj​ the substitution requires S⊆I and aik​/aij​∈Z for all k∈S, otherwise the substituted expression is not guaranteed to be integer-valued.

This reduction is sometimes called the aggregator.

Instance

\ §4.5 Substitute implied free variables
\ Achterberg et al. 2016, §4.5
\
\ x1 has explicit bounds [0, 5].
\ Inequality (c1) and the upper bound x2 <= 5 imply x1 <= 5.
\ Inequality (c2) and the lower bound x3 >= 0 imply x1 >= 0.
\ So x1's implied bounds equal its explicit bounds — it is implied free.
\
\ Equation (c3) contains x1, so the aggregator can substitute it out.

Minimize
 obj: 2 x1 + x2 + x3
Subject To
 c1: x1 - x2 <= 0
 c2: x1 - x3 >= 0
 c3: x1 + x2 + x3 = 6
Bounds
 0 <= x1 <= 5
 0 <= x2 <= 5
 0 <= x3 <= 5
End

By bound propagation on the inequality x1​−x2​≤0 together with the explicit upper bound ux2​​=5: x1​≤x2​≤5, and similarly from x1​−x3​≥0 with ℓx3​​=0: x1​≥x3​≥0. So ℓˉx1​​=0 and uˉx1​​=5 — exactly matching the explicit bounds [0,5]. Therefore x1​ is implied free, and the equation x1​+x2​+x3​=6 provides the substitution.

What fires

Substitute x1​:=6−x2​−x3​ into every other row and the objective.

The model shrinks from 3 variables / 3 constraints to 2 variables / 2 constraints, with a +12 objective constant carried forward.

Post-presolve model

\ After substituting x1 := 6 - x2 - x3 (using equation c3).
\
\ Original c1 (x1 - x2 <= 0)  becomes  6 - 2 x2 - x3 <= 0,  i.e. 2 x2 + x3 >= 6
\ Original c2 (x1 - x3 >= 0)  becomes  6 - x2 - 2 x3 >= 0,  i.e. x2 + 2 x3 <= 6
\ Original c3 is consumed by the substitution and removed.
\
\ Objective: 2 x1 + x2 + x3 = 2(6 - x2 - x3) + x2 + x3 = 12 - x2 - x3.
\ The +12 is the objective constant (uncrush offset).

Minimize
 obj: - x2 - x3 + 12
Subject To
 c1: 2 x2 + x3 >= 6
 c2: x2 + 2 x3 <= 6
Bounds
 0 <= x2 <= 5
 0 <= x3 <= 5
End

In a post-processing ("uncrush") step, the optimal x1​ is recovered from x2⋆​,x3⋆​ via x1⋆​=6−x2⋆​−x3⋆​.

Notes

  • Fill-in. Substituting xj​ out of the equation rewrites every row in which xj​ appeared as a linear combination of AiS​ and the original row coefficients. If xj​ has support in many rows, this can substantially increase the number of non-zero entries in the matrix. Practical implementations therefore guard the substitution with a fill-in budget and a numerical safeguard: if ∣aij​∣ is small relative to the largest coefficient in its row or column, the substitution is rejected to avoid magnifying round-off.
  • Numerical stability. Because the substitution divides by aij​, small pivots are dangerous in finite precision. The aggregator is a recurring source of numerical issues in presolve, and is often the first reduction to disable when a model fails to solve cleanly.
  • Integer variables. The divisibility requirement on the row's coefficients is restrictive enough that the reduction is rarely available for integer xj​ in practice; it is overwhelmingly applied to continuous variables.
  • Why "implied free" matters. Without the implied-free check, substituting xj​ would silently throw away the explicit bounds [ℓj​,uj​] — a correct optimization on the reduced model could then return a solution with xj​ outside its declared range. The check guarantees the bounds were already enforced elsewhere.
